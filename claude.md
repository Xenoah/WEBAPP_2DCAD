## 判断記録

| 日付 | 判断 | 理由 |
|------|------|------|
| 2026-07-03 | DWG ネイティブ対応を見送り、DXF R12 書き出し + R12〜2018 読み込みを互換の柱とする | DWG は非公開バイナリ形式で、合法かつオフラインで完全互換を実現する手段が存在しない。LibreDWG は GPL かつ WASM 化に難があるため Phase 7 で調査扱い |
| 2026-07-03 | 書き出しは R12 (AC1009) 固定 | R2000+ はハンドル・全テーブル・OBJECTS 辞書が必須でボイラープレートが大きい。R12 は最小構成で最も広く開ける。LWPOLYLINE は POLYLINE/VERTEX に変換(bulge 保持)し情報欠損なし |
| 2026-07-03 | コマンドエンジンはジェネレータ方式 | LINE→点→点→… のような対話フローを yield {type:'point',...} で直列に書け、AutoCAD 的なプロンプト遷移・キーワード・キャンセルを一元管理できる |
| 2026-07-03 | フレームワーク不使用（バニラ JS） | PDF エディタ (XNH_WEBAPP_pdf-editor) と同方針。ビルド不要・オフライン重視。描画は全て単一 canvas の即時再描画で React 等の恩恵が薄い |
| 2026-07-03 | 色は ACI (AutoCAD Color Index) を内部表現に採用 | DXF 62 グループコードと1対1対応し往復変換で欠損しない。256=ByLayer |
| 2026-07-05 | 拘束ソルバーは数値法（数値ヤコビアン + Levenberg-Marquardt 最小ノルム更新）を solver.js に自前実装 | 外部ライブラリ禁止方針のため。記号的手法（グラフ分解）より実装が単純で、18 種の拘束を残差関数の追加だけで拡張できる。最小ノルム更新 dx=Jᵀ(JJᵀ+λI)⁻¹(−r) により「現在の形状から最も近い解」へ動き、AutoCAD 的な直感に合う |
| 2026-07-05 | 過拘束=ヤコビアンのランク解析（式数−ランク=冗長数、変数−ランク=自由度）、矛盾=非収束で判定 | 数値法で判別可能な範囲として妥当。冗長でも整合する場合は警告（⚠過拘束）、非収束は矛盾疑い（✖）としてエラー表示し図形は動かさない |
| 2026-07-05 | 拘束対象は line / circle / arc / point の 4 型に限定 | pline を変数化すると頂点数分 DOF が爆発し UI も複雑化。必要なら EXPLODE で対応 |
| 2026-07-05 | 寸法は非関連付け（作成時のスナップショット）とし、図形追従は寸法拘束に担わせる | 関連付け寸法は参照追跡の実装コストが高い。パラメトリック機能と役割分担する |
| 2026-07-05 | 寸法の DXF 出力は線分+文字への分解方式 | R12 の DIMENSION 実体+匿名ブロック生成はボイラープレートが大きい。他 CAD での見た目互換を優先し、完全な状態は JSON 側で保持 |
| 2026-07-05 | 画像 dataURL は undo スナップショットに含めず S.imageData に分離保持（エンティティはキー参照） | スナップショット方式 undo（最大100件）で数 MB の dataURL を毎回 clone するとメモリが破綻するため |
| 2026-07-05 | JSON プロジェクト形式（app タグ + version 付き）を正式な保存形式に追加 | 寸法スタイル・拘束・画像は DXF R12 で表現不能。DXF は交換用、JSON は作業状態の完全保存用と位置付け |
